# Contributing

Thanks for taking the time to improve Linka. Keep contributions focused, practical, and easy to review.

## Issues

Open an issue for bugs, setup problems, or feature ideas. Include:

- What you expected to happen.
- What actually happened.
- Your Windows version, Node.js version, and whether you used the packaged app or development mode.
- Screenshots or logs when they help explain the problem.

## Pull Requests

Before opening a pull request:

- Keep the change scoped to one bug fix or feature.
- Run `npm run lint`.
- Describe what changed and how you tested it.
- Do not commit generated output such as `dist`, `dist_electron`, `node_modules`, or native `bin/obj` folders.

Small, clear pull requests are easiest to review and merge.
