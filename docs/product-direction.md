# Linka Product Direction

## Primary product

- Remote ergonomic control
- Remote presence on the same local network
- Bridge for lightweight local text, image, and file handoff
- Fast, dependable remote computing without cloud dependency

## Secondary / experimental

- Hermes integration
- Teach mode
- Workflow memory and automation experiments

## Guardrails

- Core remote control and Bridge flows stay stable first.
- Hermes features should remain isolated and reversible until they prove product value.
- Avoid protocol churn unless it fixes a concrete reliability or security problem.
- Prefer incremental modularization over framework rewrites.
